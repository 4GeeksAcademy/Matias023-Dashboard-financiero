_Financial metrics dashboard with a React + TypeScript frontend and a FastAPI backend._

## Project overview

This repository contains a financial dashboard that currently renders:

- KPI cards for total income, total outcome, profit, and profit margin.
- Two charts: monthly income vs outcome, and monthly profit margin.

The data displayed is **simulated mock financial data** generated by the backend at request time. It is not connected to a real accounting source or persistent database.

Current verifiable scope:

- Frontend fetches `/api/metrics`.
- Backend exposes multiple read-only analytics endpoints.
- Docker Compose runs both services together for local development.

## Technology overview

- React + TypeScript for frontend UI and data processing.
- Vite for frontend dev server and build tooling.
- FastAPI + Python for backend API endpoints.
- Docker Compose for local multi-service orchestration.
- Testing:
   - Frontend: Vitest.
   - Backend: pytest with FastAPI TestClient.

## Running the project

```bash
docker compose up --build
```

The frontend uses the Vite proxy for `/api` by default, so no extra environment variables are required for the default local setup.

- Frontend: http://localhost:5173
- API documentation: http://localhost:8000/docs
- Health endpoint: http://localhost:8000/health

Note: `http://localhost:8000` can return `404 Not Found` because there is no `/` route defined in the backend.

## Application flow

1. Vite serves the frontend.
2. React mounts the app from `frontend/src/main.tsx`.
3. `frontend/src/App.tsx` requests `/api/metrics`.
4. Vite proxy forwards `/api` requests to the backend (`http://backend:8000` in Docker network).
5. FastAPI generates and returns simulated financial movements.
6. Frontend computes KPIs and monthly series in `frontend/src/lib/financial-utils.ts`.
7. Dashboard components render cards and charts.

## Available API capabilities

- Health:
   - `GET /health`
- Metrics data and filtering:
   - `GET /api/metrics`
   - `GET /api/metrics/b2b`
   - `GET /api/metrics/b2c`
   - `GET /api/metrics/facets`
- Aggregations and analytics:
   - `GET /api/metrics/summary`
   - `GET /api/metrics/categories/top`
   - `GET /api/metrics/comparison`
   - `GET /api/metrics/alerts`

## Testing

- Frontend tests:

```bash
docker compose exec -T frontend npm run test
```

- Backend tests:

```bash
docker compose exec -T backend pytest -q
```

## Current limitations

- Uses simulated financial data generated in backend code.
- No real persistence layer (no database integration).
- No CI workflow files are currently visible in this repository.
- Python dependencies in `backend/requirements.txt` are not pinned to exact versions.
- Main KPI and monthly chart calculations are performed in the frontend.
